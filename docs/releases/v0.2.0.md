# Engineering Ownership v0.2.0

v0.2 turns Engineering Ownership into an immediately usable ownership workflow
without expanding it into a full development framework.

## Highlights

- One `$engineering-ownership` router for setup, start, resume, pre-merge
  checks, handoff, and optional study.
- Dated, titled decision records and upward-only R2/R3 escalation.
- `engineering doctor`, `engineering refs check`, and Git-ignored saved
  handoffs.
- Declared risk remains a floor even when the current diff looks lower risk.
- Optional SessionStart and Stop reminders for Codex and Claude. They are no-op
  by default, read-only when enabled, non-blocking, and offline.
- English and Korean first-work tutorials plus integration guidance for
  existing planning, TDD, QA, spec, and session-state frameworks.

## Compatibility

Contract v2 and evidence v1 remain readable without migration. Existing record
filenames are unchanged. New evidence adds an optional title, and new records
use dated headings.

## Verification

The release is tested on Python 3.11–3.13 and macOS/Ubuntu in CI. Release assets
include a deterministic plugin ZIP, SHA-256 checksum, and GitHub build
provenance.
